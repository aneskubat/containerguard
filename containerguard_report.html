Dockerfile — nalazi
CRITICAL DF001 Kontejner se pokreće kao root korisnik
Dockerfile
Pokretanje kontejnera kao root je ozbiljan sigurnosni propust. Ako napadač kompromituje aplikaciju, automatski ima root pristup kontejneru, što olakšava container escape napade.
Rješenje: Dodaj 'USER appuser' na kraj Dockerfile-a (prethodno kreiraj korisnika s RUN adduser).
HIGH DF002 Koristi se 'latest' tag za base image
Dockerfile · Linija 5
Tag 'latest' nije fiksna verzija — mijenja se. Danas je sigurna slika, sutra može sadržavati ranjivosti. Nemoguće je reproducirati isti build.
Rješenje: Koristi specifičan tag, npr. FROM python:3.11.9-slim umjesto FROM python:latest
CRITICAL DF003 Hardcoded lozinka ili API ključ u ENV
Dockerfile · Linija 8
Tajni podaci hardcoded u Dockerfile postaju dio Docker image sloja. Svako ko skine image može ih pročitati komandom 'docker history'.
Rješenje: Koristi Docker Secrets ili environment varijable koje se proslijeđuju pri pokretanju.
MEDIUM DF004 Koristi se ADD umjesto COPY
Dockerfile · Linija 12
ADD instrukcija ima skrivene mogućnosti: automatski raspakuje arhive i može preuzimati fajlove s URL-ova. COPY je eksplicitniji i sigurniji.
Rješenje: Zamijeni ADD s COPY za kopiranje lokalnih fajlova.
MEDIUM DF005 Nedostaje HEALTHCHECK instrukcija
Dockerfile
Bez HEALTHCHECK instrukcije, Docker ne zna je li aplikacija funkcionalna. Kontejner može biti 'running' ali aplikacija može biti crashala.
Rješenje: Dodaj HEALTHCHECK instrukciju, npr: HEALTHCHECK --interval=30s CMD curl -f http://localhost:8080/health || exit 1
LOW DF006 apt-get bez --no-install-recommends
Dockerfile · Linija 17
Instaliranje paketa bez --no-install-recommends dovodi do instalacije nepotrebnih paketa koji povećavaju površinu napada.
Rješenje: Koristi: RUN apt-get install -y --no-install-recommends
docker-compose.yml — nalazi
CRITICAL DC001 Privileged mod je uključen
docker-compose.yml
privileged: true daje kontejneru gotovo isti pristup hostu kao root korisnik. Napadač može lako izvesti container escape napad.
Rješenje: Ukloni 'privileged: true'. Koristi 'cap_add' samo za specifične capabilities.
CRITICAL DC002 Docker socket je montiran u kontejner
docker-compose.yml
Montiranje /var/run/docker.sock daje kontejneru punu kontrolu nad Docker daemonom. Napadač može kreirati privilegovane kontejnere i preuzeti cijeli host.
Rješenje: Nikad ne montiraj Docker socket u produkcijske kontejnere.
MEDIUM DC004 Nedostaju resource limits (CPU/memorija)
docker-compose.yml
Bez resource limits, jedan kontejner može potrošiti sve resurse hosta i srušiti ostale servise.
Rješenje: Dodaj limits: deploy: resources: limits: cpus: '0.5' memory: 512M
LOW DC005 Container restart policy nije definisan
docker-compose.yml
Bez restart policy, kontejner se ne pokreće automatski nakon pada sistema.
Rješenje: Dodaj 'restart: unless-stopped' za produkcijske servise.
