CRITICAL K8S001 Container se pokreće kao root (runAsRoot)
Pokretanje kontejnera kao root u Kubernetes okruženju je kritičan propust. U slučaju kompromitacije, napadač dobiva root pristup kontejneru što značajno olakšava container escape napade prema host sistemu.
Rješenje: Dodaj securityContext: runAsNonRoot: true runAsUser: 1000
CRITICAL K8S002 Privilegovani kontejner (privileged: true)
Privilegovani kontejner ima pristup svim Linux capabilities i može direktno pristupati host uređajima. Ekvivalentno pokretanju procesa s root pravima direktno na hostu.
Rješenje: Ukloni 'privileged: true' iz securityContext.
HIGH K8S003 Nedostaju resource limits
Bez definisanih CPU i memorijskih limita, jedan Pod može potrošiti sve resurse noda i srušiti ostale servise — resource exhaustion napad. Kubernetes scheduler također ne može optimalno rasporediti Podove.
Rješenje: Dodaj resources: limits: cpu: '500m' memory: '512Mi' requests: cpu: '250m' memory: '256Mi'
HIGH K8S004 Koristi se 'latest' tag za container image
Tag 'latest' nije deterministički — svaki deployment može pokrenuti drugačiju verziju image-a. Ovo otežava audit, rollback i može uvesti ranjivosti bez znanja operatora.
Rješenje: Koristi specifičan tag, npr. nginx:1.25.3 umjesto nginx:latest
HIGH K8S005 allowPrivilegeEscalation nije onemogućen
Bez 'allowPrivilegeEscalation: false', proces unutar kontejnera može steći više privilegija od roditeljskog procesa kroz setuid/setgid mehanizme, što olakšava privilege escalation napade.
Rješenje: Dodaj u securityContext: allowPrivilegeEscalation: false
MEDIUM K8S006 Nedostaju liveness i readiness probe
Bez health proba, Kubernetes ne može automatski detektovati i restartati nezdrave kontejnere. Aplikacija može biti u neispravnom stanju ali Kubernetes nastavlja slati saobraćaj na nju.
Rješenje: Dodaj livenessProbe i readinessProbe u container spec.
MEDIUM K8S007 Root filesystem nije read-only
Bez readOnlyRootFilesystem: true, napadač koji kompromituje kontejner može pisati na filesystem — instalirati alate, modifikovati konfiguracije ili persistovati malware.
Rješenje: Dodaj u securityContext: readOnlyRootFilesystem: true
MEDIUM K8S008 Hostnetwork ili hostPID je omogućen
hostNetwork: true ili hostPID: true uklanjaju mrežnu/procesnu izolaciju između kontejnera i hosta. Napadač može vidjeti sve mrežne interfejse i procese na host sistemu.
Rješenje: Ukloni 'hostNetwork: true' i 'hostPID: true' iz Pod spec-a.
CRITICAL K8S010 Secrets u environment varijablama (plaintext)
Definisanje tajnih podataka direktno u env varijablama Pod-a znači da su vidljivi svakome s pristupom kubectl describe. Kubernetes Secrets objekat pruža bolju kontrolu pristupa.
Rješenje: Koristi secretKeyRef umjesto direktnih vrijednosti: env: - name: DB_PASSWORD valueFrom: secretKeyRef: name: myapp-secrets key: db-password
